# Проект: Автоматизация списка кредиторов для банкротства

## Что делает этот проект
Извлекает данные о кредиторах из кредитной истории НБКИ и исполнительных производств,
сопоставляет их и формирует готовый список кредиторов для заявления о банкротстве.

## Входные файлы (input/)
- `credit_history.pdf` — кредитная история из НБКИ (PDF)
- `enforcement_proceedings.docx` — исполнительные производства с Госуслуг (DOCX)
- `creditors_template.docx` — эталонный заполненный документ Елены (только для сверки)

## Выходные файлы (output/)
- `creditors.json` — список кредиторов из НБКИ
- `proceedings.json` — список исполнительных производств
- `matched.json` — итоговый сматченный список
- `final_creditors_list.docx` — готовый документ для подачи

## Правила извлечения данных

### Из НБКИ (кредитная история):
- Если была переуступка долга — берём нового кредитора (из "Сведения о приобретателе прав")
- Сумма берётся как текущий остаток долга, не первоначальная сумма кредита
- Для каждого кредитора нужно: наименование, ИНН, адрес, сумма долга

### Из исполнительных производств:
- Если "исполнительский сбор" — идёт в Раздел 2 списка кредиторов
- Если взыскатель совпадает с кредитором из Раздела 1 — добавляем ИП к этой строке
- Если взыскатель ИФНС — вносим как "налог" в Раздел 2

## Формат JSON для кредиторов
```json
{
  "name": "ООО МКК Срочноденьги",
  "inn": "5260271530",
  "ogrn": "1105260001857",
  "address": "",
  "debt_total": 9600.00,
  "contract_number": "44444/4654938",
  "contract_date": "25-10-2023",
  "proceedings": []
}
```

## Важно
- Не изменяй входные файлы
- Все суммы в рублях, числа (не строки)
- При неоднозначном матчинге — сохраняй в отдельный список `unmatched.json` для ручной проверки
